# 系统架构文档

## 概述

数字遗产管家是一个基于门限密码学和去中心化触发条件的数字资产继承系统。本文档详细描述系统的整体架构、技术选型和设计决策。

## 核心模块

### 1. 密码学模块

#### Shamir秘密共享 (SSS)

**功能：**
- 将主密钥分割成n个份额
- 任意t个份额可恢复主密钥
- 少于t个份额无法获得任何信息

**实现：**
```typescript
class ShamirSecretSharingImpl {
  split(secret: string, n: number, t: number): Share[]
  combine(shares: Share[]): string
  generateCommitment(share: Share): string
  verifyCommitment(share: Share, commitment: string): boolean
}
```

**安全特性：**
- 信息论安全
- 防止份额伪造
- 支持份额验证

#### PVSS (可验证秘密共享)

**功能：**
- 为每个份额生成承诺
- 零知识证明验证份额真实性
- 防止恶意监护人提交假份额

**实现：**
- 使用椭圆曲线生成密钥对
- SHA-256哈希生成承诺
- 公开验证承诺

#### 门限签名

**功能：**
- 多方协作生成签名
- 无需重构私钥
- 直接在链上执行交易

**优势：**
- 私钥永不暴露
- 更高的安全性
- 支持区块链资产直接转移

### 2. 业务逻辑模块

#### 遗产计划管理

**功能：**
- 创建遗产计划
- 更新计划信息
- 删除计划
- 查询计划详情

**数据结构：**
```typescript
interface LegacyPlan {
  id: string
  name: string
  assets: any[]
  guardians: any[]
  threshold: number
  totalShares: number
  triggerMode: 'time' | 'consensus' | 'hybrid'
  timeLock: number
  masterKey: string
  shares: Share[]
  encryptedAssets: string
  status: 'active' | 'triggered' | 'completed'
}
```

#### 继承流程管理

**功能：**
- 发起继承请求
- 收集监护人份额
- 验证份额有效性
- 恢复数字资产

**流程：**
1. 继承人发起请求
2. 监护人提交份额
3. 系统验证份额
4. 恢复主密钥
5. 解密资产包
6. 转移资产

#### 触发条件验证

**触发模式：**

1. **时间锁模式**
   - 设定固定时间
   - 时间到期自动解锁
   - 适用于养老规划

2. **社会共识模式**
   - 需要t位监护人确认
   - 快速响应
   - 适用于意外身故

3. **混合模式**
   - 时间锁 + 监护人确认
   - 双重保险
   - 适用于高价值资产

### 3. 智能合约模块

#### LegacyContract

**主要功能：**
- 存储遗产计划
- 验证触发条件
- 记录份额提交
- 管理计划状态

**关键函数：**
```solidity
function createPlan(...) public
function triggerPlan(string memory planId) public
function submitShare(...) public onlyGuardian(planId)
function verifyCommitment(...) public view returns (bool)
function completePlan(string memory planId) public
```

**安全特性：**
- 访问控制（onlyOwner, onlyGuardian）
- 状态转换验证
- 防止重入攻击
- Gas优化

```
  ↓ POST /api/plans

  │
  ├─ 4. 生成主密钥
  ├─ 5. 分割主密钥 (Shamir SSS)
  ├─ 6. 加密资产包

  ↓ 存储计划

  │
  ↓ 返回计划信息

  │
  └─ 8. 显示计划详情
```

```

  │
  ├─ 1. 发起继承请求
  │
  ↓ POST /api/inheritance/initiate

  ├─ 2. 验证触发条件
  ├─ 3. 创建继承请求
  │
  ↓ 通知监护人

  │
  ├─ 4. 提交份额

  │
  └─ 9. 接收资产
```

## 安全架构

### 1. 密码学安全

- **主密钥管理**：主密钥永不存储，只以份额形式存在
- **份额加密**：每个份额用监护人公钥加密
- **零知识证明**：PVSS确保份额真实性
- **门限签名**：私钥永不重构

### 2. 网络安全

- **HTTPS**：所有通信使用TLS加密
- **CORS**：严格的跨域资源共享策略
- **速率限制**：防止DDoS攻击
- **输入验证**：所有输入都经过验证

### 3. 智能合约安全

- **访问控制**：严格的权限管理
- **重入保护**：使用Checks-Effects-Interactions模式
- **整数溢出**：Solidity 0.8+自动检查
- **Gas优化**：减少不必要的操作

### 4. 数据安全

- **加密存储**：所有敏感数据加密存储
- **最小权限**：只授予必要的权限
- **审计日志**：记录所有关键操作
- **备份策略**：定期备份数据

## 总结

数字遗产管家的架构设计遵循以下原则：

1. **安全性优先**：密码学保障，多层防护
2. **去中心化**：减少单点故障
3. **可扩展性**：支持水平和垂直扩展
4. **用户体验**：简洁直观的界面
5. **开源透明**：代码可审计，社区可参与

该架构为数字遗产继承提供了安全、可靠、可扩展的解决方案。
